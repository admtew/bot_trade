# UTM Яндекс Директ → AmoCRM через Битрикс

## Как работает

1. Пользователь переходит по рекламе: `site.ru/?utm_source=yandex&utm_medium=cpc&...`
2. JS скрипт сохраняет UTM в cookies (30 дней)
3. При отправке любой формы — UTM подставляются в скрытые поля
4. Данные уходят в AmoCRM как доп. поля сделки

## Установка

### Шаг 1: JS скрипт на сайт

Вставить `utm-tracker.js` перед `</body>` на всех страницах.

В Битрикс: **Настройки → Настройки продукта → Сайты → Шаблон сайта → footer.php**

```html
<script src="/local/js/utm-tracker.js"></script>
```

Или через включаемую область / header.php.

### Шаг 2: Создать поля в AmoCRM

В AmoCRM → Сделки → Настроить → добавить поля:
- `utm_source` (текст)
- `utm_medium` (текст)
- `utm_campaign` (текст)
- `utm_term` (текст)
- `utm_content` (текст)

### Шаг 3: Связать формы с AmoCRM

Зависит от того, как сейчас устроена интеграция:

**Вариант А: Битрикс CRM-формы + стандартная интеграция AmoCRM**
— Нужно замапить скрытые поля формы на поля AmoCRM в настройках интеграции.

**Вариант Б: Webhook / свой обработчик**
— UTM поля придут в POST-запросе вместе с остальными данными формы.
— В обработчике нужно передать их в AmoCRM API.

**Вариант В: Прямая отправка через AmoCRM API**
— Использовать `window.UTMTracker.getUtm()` для получения значений и добавления в тело запроса.

### Шаг 4: Проверка

1. Открыть сайт с UTM: `?utm_source=test&utm_medium=cpc`
2. Проверить cookies в DevTools (Application → Cookies)
3. Отправить форму
4. Проверить что в AmoCRM сделка содержит UTM поля

## Для AJAX-форм (ручная передача)

```js
var utm = window.UTMTracker.getUtm();
// utm = { utm_source: 'yandex', utm_medium: 'cpc', ... }

// Добавить в данные формы перед отправкой:
Object.assign(formData, utm);
```
